# myroom-vr — 書斎と縁側と庭

Meta Questのブラウザから入れる、和モダンの自分の部屋。WebXR + Three.js + Vite + TypeScript。

**▶ 公開URL: https://lalalatang.github.io/myroom-vr/** (Questブラウザで開いて「ENTER VR」)

- 書斎(文机・本棚・掛け軸・行灯・ラジオ・畳)→ 縁側(風鈴・障子)→ 庭(飛び石・池・楓・鹿威し・焚き火)がシームレスにつながる約20m四方の空間
- 朝/昼/夕/夜の切替(HDRI・ライト連動)、晴れ/雨の天候、各種ギミック
- アセットはすべてCC0(Poly Haven / ambientCG)。一覧は [ASSETS.md](ASSETS.md)

## 起動(ローカル開発)

```bash
npm install
npm run fetch-assets   # CC0アセット(HDRI・テクスチャ)をダウンロード(初回のみ)
npm run dev            # https://localhost:5173 (自己署名HTTPS。WebXRに必須)
```

デスクトップブラウザでの操作(開発検証用フォールバック):

| 操作 | キー |
|---|---|
| 移動 | クリックで操作開始 → **WASD**(Shiftで速く) |
| 視点 | マウス |
| 仕掛けの操作 | 画面中央を対象に合わせて**左クリック** |
| 時間帯 | **1**=朝 **2**=昼 **3**=夕 **4**=夜 |
| 天候 | **R** で晴れ⇔雨 |

## 【帰宅後にやること】Quest実機での確認手順

開発中はQuest実機が使えなかったため、実機確認は以下の手順で行ってください。

1. **同じWi-Fi**にPCとQuestを接続し、PCで `npm run dev` を実行。ターミナルに表示されるLAN URL(例: `https://192.168.0.49:5173`)を控える
2. QuestのブラウザでそのURLを開く。自己署名証明書の警告が出たら「詳細設定」→「アクセスする(続行)」を選ぶ
3. 画面下の「**ENTER VR**」ボタンで没入モードに入る
4. P0チェックリスト:
   - [ ] **移動**: 左スティック前倒しで放物線の照準 → 離すとテレポート。右スティック左右でスナップターン45°
   - [ ] **昼夜**: 左手首のパネルで朝/昼/夕/夜を切替(トリガーでボタンを撃つ)
   - [ ] **照明**: 行灯・デスクランプをレイでポイントしてトリガー → 個別に点灯/消灯
   - [ ] **ラジオ**: 文机の上のラジオをトリガー → 再生/停止
   - [ ] **障子**: 書斎⇔縁側の障子をトリガー → スライド開閉
   - [ ] 体感が滑らか(72fps目安。カクつく場合はブラウザの他タブを閉じる)
5. GitHub Pages版(デプロイ済みなら)はURLを開くだけ。HTTPSが正規証明書なので警告なしで入れます

## ラジオ音源の差し替え方法

`public/audio/` に **mp3(ogg/m4a/wavも可)を置くだけ**です。ファイル名順に連続再生されます。

```bash
cp ~/Podcasts/ep01.mp3 public/audio/
# devサーバならリロードで反映。本番はビルドし直して再デプロイ
```

プレイリスト(`audio/playlist.json`)はViteが自動生成するため、JSONの手編集は不要です。
音源が1つもない場合は、プロシージャル合成の「深夜ラジオ風」音声が流れます。

## 短歌(掛け軸)の差し替え方法

`public/texts/tanka.json` を編集します。形式:

```json
{
  "tanka": [
    { "lines": ["秋の田の", "かりほの庵の", "苫をあらみ", "わが衣手は", "露にぬれつつ"], "author": "天智天皇" }
  ]
}
```

掛け軸をクリック(VRではトリガー)すると次の一首に切り替わります。

## ビルド・デプロイ

```bash
npm run build     # 型チェック + dist/ 生成
```

(`.github/workflows/deploy.yml`)。

## 自動検証(デスクトップ)

```bash

node scripts/verify-desktop.mjs /tmp
# ロード・WASD移動・時間帯/天候切替・ドローコール/三角形数をヘッドレスChromeで検証し、スクリーンショットを保存
```

## アーキテクチャ

- `src/core/` — モジュール間契約(types.ts)・レイアウト規約(layout.ts)・状態(state.ts)・イベントバス(events.ts)・インタラクション登録(registry.ts)
- `src/world/` — ジオメトリ(家屋・庭・家具)とPBRマテリアル
- `src/systems/` — XRコントローラー・テレポート・デスクトップ操作・照明(昼夜)・天候・空間オーディオ
- `src/interactions/` — 各ギミック(障子・行灯・ラジオ・掛け軸・鹿威し・焚き火・風鈴・手首パネル)

設計上の仮定・決定は [DECISIONS.md](DECISIONS.md) を参照。
