### XR入室パスの静的検証(実機の代替)
コードレビューで以下を確認:
- HTTPS前提(dev=basic-ssl / 本番=Pages)、`renderer.xr.enabled=true`、`VRButton`(`isSessionSupported('immersive-vr')`→`requestSession`)の標準経路
- コントローラーは `renderer.xr.getController/getControllerGrip` をプレイヤーリグの子に追加(テレポートでリグごと移動する構成)、handedness判定は`connected`イベント
- テレポート/スナップターンは`session.inputSources`のgamepad軸(Questのaxes[2,3])を毎フレームポーリング
- AudioContextはユーザー操作/`sessionstart`でresume(自動再生制限対応)
- foveation=1.0、影はdirectional 1灯のみ・1024px、ポストプロセスなし(§5準拠)
- ヘッドレス環境ではVRButtonが正しく「VR NOT SUPPORTED」表示になることをスクリーンショットで確認(=WebXR分岐が例外なく通っている)

## 帰宅後にユーザーがやる3手順

1. **QuestでPages版を開く**: Questのブラウザで https://lalalatang.github.io/myroom-vr/ → 「ENTER VR」→ READMEのP0チェックリスト(移動/昼夜/照明/ラジオ/障子)を実機確認。
   カクつく場合は他タブを閉じる。それでも重ければ報告を(影解像度・雨粒数を下げる調整余地あり)
2. **ラジオに手持ちの音源を入れる**: `public/audio/` にmp3をコピー → `git add -A && git commit -m "add audio" && git push`(自動で再デプロイ)。
   ※公開リポジトリなので、**公開してよい音源のみ**置くこと(著作権のある番組はプライベート化するか、ローカル`npm run dev`でのみ使用)
3. **LAN開発モードも一度確認**: `npm run dev` → 表示されるLAN URL(例 https://192.168.0.49:5173)をQuestで開き、証明書警告を「続行」で通過して入室できることを確認(今後の開発イテレーション用)
