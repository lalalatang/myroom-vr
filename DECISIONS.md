# DECISIONS.md — 仮定と決定の記録

ユーザー不在のため、不明点は以下の仮定を置いて進めた(指示に基づく)。

## D1: Quest実機確認の代替(ユーザー指示)
GOAL §9-2「まずQuest実機で入れることを確認」は実機が手元にないため実施不可。代替として
(a) デスクトップブラウザでのWASD+マウス動作確認、(b) WebXR入室パスのコードレビューによる静的検証で通す。
実機確認手順はREADMEに「帰宅後の宿題」として記載。

## D2: GitHubリポジトリを新規作成(public)
GitHub Pagesの無料利用はpublicリポジトリが条件のため、`lalalatang/myroom-vr` をpublicで作成する。
中身はCC0アセットと自作コードのみでライセンス問題なし。gh認証は確認済み。

## D3: テクスチャはKTX2/Basisではなく1K JPGを採用
GOAL §5は「最大2K・KTX2/Basis圧縮」だが、KTX2変換にはtoktx等の外部ツール導入とパイプライン構築が必要で、
今回の規模(小空間・テクスチャ数点)では1K JPGでもVRAM・ロード予算内に収まると判断。
KTX2化は将来最適化としてREPORT.mdに提案として記載。

## D4: 効果音はWebAudioでプロシージャル合成
CC0の効果音アセットは調達先の確実性が低い(Freesoundはライセンス混在・要認証)。
鹿威し・風鈴・雨・焚き火・クリック音などは実行時にWebAudioで合成する(=外部アセット不要、ライセンス問題ゼロ、転送量ゼロ)。
ラジオは仕様どおり `public/audio/*.mp3` を順次再生。**Viteプラグインで playlist.json を自動生成**するため、
mp3を置くだけで差し替え可能(JSON手編集不要)。mp3が0件の場合はプロシージャルなラジオノイズ+環境音で代替再生。

## D5: HDRIはPoly Havenの2K HDRを4枚(朝/昼/夕/夜)
P0-2が「朝/昼/夕/夜」の4状態のため4枚。2K HDRは1枚1〜2MB程度でリポジトリ管理可能。
dl.polyhaven.org が一時的に521を返したため、fetch-assets.sh はAPI経由でURL解決+リトライを実装。
取得失敗時もアプリはプロシージャル空(グラデーション)で動作するようフォールバックを実装。

## D6: 座標系・寸法の規約(core/layout.ts に固定)
北(-Z)に家屋、南(+Z)に庭。書斎の床(畳)は地面+0.4m(縁側らしさのため段差を採用)。
テレポート・WASDとも接地面へのレイキャストで高さ追従するため往来は問題なし。天井高は床から2.4m。
スポーンは土間(書斎西側の三和土、y=0)。全体は20m×20m以内。

## D7: Vite base は './'(相対パス)
GitHub Pages のプロジェクトサイト(/myroom-vr/)とローカルの両方で同一ビルドが動くようにするため。

## D8: 実装分担
GOAL §7どおり。世界ジオメトリ/アセット取得/UI/短歌=Sonnet、XR入力/照明/天候/音響=Opus、
骨格(contract定義・main配線)・統合・最適化=Fable本体。モジュール契約は src/core/types.ts に集約。

## D9: 統合時の修正(Fable統合パスで発見・決定)
- **materials.ts に tint オプション追加**: three.js の material.color はテクスチャに乗算されるため、
  フォールバック用の暗い color を残すとテクスチャ読込後に黒ずむ。読込後は白(または明示 tint)にリセットする仕様にした。
- **石テクスチャを Rock035 → Rock023 に変更**: Rock035 はほぼ黒のスレートで、飛び石・沓脱石が黒い塊に見えたため、
  明るい灰色の Rock023(同じく ambientCG / CC0)に差し替え。ASSETS.md も更新済み。
- **板天井・軒天を追加**: 片流れ屋根の裏面が室内から見えるとモアレ状に汚いため、書斎・土間に板天井、縁側に軒天を張った。
- **HDRI環境光強度(envIntensity)を時間帯プロファイルに追加**: 追加しないと夜でも室内が昼のように明るい。
  夜0.1/夕0.5/朝0.85/昼1.0でクロスフェード。雨天減光(×0.6)も lighting に一元化し、weather との二重管理を解消。
- **掛け軸の短歌パネルの向きを修正**: userData.faceNormal はワールド法線だが、パネルは掛け軸グループの子のため
  ローカル法線に変換してから向きを合わせるようにした(修正前は90°ずれて壁から垂直に浮いていた)。
- **共有マテリアルの分離**: 行灯・デスクランプの発光部が同一の washi マテリアルを共有しており、
  片方の点灯で両方光るバグを修正(各自の materialインスタンスに)。畳縁も艶消し濃紺の専用マテリアルに分離。

## D10: 検証方法(Quest実機なしの代替、ユーザー指示)
- デスクトップ検証は puppeteer-core + システムChrome(ヘッドレス)で自動化(scripts/verify-desktop.mjs)。
  結果: ページロード/コンソールエラー0/WASD移動2.28m/時間帯・天候切替/ドローコール49/三角形11,154(いずれも予算内)。
- XR入室パスは静的コードレビューで検証(詳細はREPORT.md)。実機確認手順はREADMEに記載。

npm依存はすべてOSS無料。GitHub Pages無料枠。外部有料API・キー発行なし。
